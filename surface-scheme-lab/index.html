Layering rules

  • default is the base and should not stack on itself.
  • default can contain subtle, or raised for emphasis.
  • default on sunken is a preferred combination.
  • subtle should not sit directly on sunken.

Evaluation focus

The goal is not to prove every combination possible. It is to find the combinations that still feel legible and intentional when color alone carries the hierarchy.

Usage Rules

One table to formalize depth levels and allowable combinations

In a borderless system, the table itself is a default card and its header and footer become subtle. That keeps the pattern consistent with the same rules the rest of the page follows.

Level Purpose Allowed on Avoid on Typical examples
Sunken Structural field that frames other surfaces Page stripes, content rails, footer zones Inside default cards as a subsurface Section scaffolding, wide layout bands
Default Primary reading and interaction surface On sunken parents, or as the page base On other default parents Main cards, tables, content wells, body canvas
Subtle Quiet grouping and secondary regions Inside default cards, on default bands Directly on sunken Insets, grouped metadata, table chrome, side panels
Raised Attached emphasis and quiet fills Inside default cards or subtle cards As a sibling card beside sunken or subtle without a shared parent Card headers, quiet action buttons, faint support
Content hierarchy follows the same logic: faint only on default or raised, fills always pair with matching on-fill tokens, and alt families stay contained unless the whole band is explicitly alt.
